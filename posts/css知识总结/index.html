一、文档流与盒模型

1.文档流

display 流动方向 宽度 高度
inline 左-右 最右折行 内部 inline 元素的和，不能 width 指定 由 line-height 间接确定与 height 无关
block 上-下 默认自动计算宽度，可用 width 指定 由内部文档流元素决定，可设 height
inline-block 左-右 结合前两者特点，可用 width 指定 与 block 类似，可设 height

2.盒模型

content-box border-box
内容边界为盒子边界 边框是盒子边界
width=内容宽度 width=内容宽度+padding2+border2

CSS 布局

float 布局，兼容 IE，专为 IE 准备。

步骤：1.子元素加上 float:left 和 width 2.父元素上加.clearfix

.clearfix:after {
  content: "";
  display: block;
  clear: both;
}

注：手机页面不用 float 布局。float 要程序员自己计算宽度，不灵活，但足以应付 IE。

CSS 定位

布局是屏幕平面上的，定位是垂直于屏幕的

背景的范围是 border 外沿围成的区域，background 在文字后面，可用 border 半透明及代码验证。 一个 div 的分层由上到下为：

  1. 内联元素
  2. 浮动元素
  3. 块级子元素
  4. border
  5. background (图略，自行脑补) 浮动元素脱离文档流其实就是浮起来一点点。 position属性：
  • static 默认值，待在文档流里。
  • relative 相对定位，升起来，但不脱离文档流。
  • absolute 绝对定位，定位是其先祖里的非 static 元素。
  • fixed 固定定位，定位基准是 viewport，移动端 bug 多。
  • sticky 粘滞定位 （导航/广告）兼容性差。

z-index

  1. auto 默认值，不创建新的层叠上下文。
  2. 设置值为 0/1/2 等创建新的层叠上下文。
  3. -1/-2 值降低同一层叠上下文中的层级。 学会管理 z-index

CSS 动画

###　浏览器渲染过程：

  1. 根据 HTML 构建 HTML 树（DOM）
  2. 根据 CSS 构建 CSS 树（CSSOD）
  3. 将两棵树合并成一颗渲染树（render tree）
  4. Layout 布局（文档流，盒模型，计算大小和位置）
  5. Paint 绘制（把边框颜色，文字颜色，阴影等画出来）
  6. Compose 合成（根据层叠关系展示画面）

一般以 JS 来更新样式，分为三种不同的渲染方式：

  1. JS/CSS > 样式 > 布局 > 绘制 > 合成 （全走）
  2. JS/CSS > 样式 > 绘制 > 合成 （跳过 Layout）
  3. JS/CSS > 样式 > 合成 (跳过 Layout 和 Paint)

transform 属性属于第三种，性能优于前两者。 每个属性触发哪些流程参考CSSTriggers 网站

transition 过渡：

作用：补充中间帧 语法：

  • transition:属性名 时长 过渡方式 延迟 transition: left 200ms linear
  • 可用逗号分隔两个不同的属性 transition: left 200ms,top 400ms
  • 可用 all 代表所有属性 transition: all 200ms
  • 过渡方式有：line | ease | ease-in | ease-out 等

注意不是所有属性都能过渡，如 display:none=>block 就没法过渡，一般用 visibility:hidden=>visible代替

animation 过渡：

animation: 动画名 | 时长 | 过渡方式 | 次数 | 方向 |
@keyframes 动画名 {
    0% {top: 0;left: 0}
    50% { transform:translateX(100PX);}
    你需要的节点位置% {transform:translateX(100px);transform:translateY(100px);}
    100% {transform:translate(100PX,100px,200px);}
}

注意：CSS 的属性非常多，需要不断练习使用来掌握它的使用技巧，练习中不会的 Google 搜索“关键字”+MDN 并学会看 MDN 的文档。
